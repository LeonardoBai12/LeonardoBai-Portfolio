This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
make serve      # Serve locally at http://localhost:4000 with live reload
make build      # Build static site to _site/
make articles   # Manually sync Medium articles (runs scripts/fetch_medium.py)
```

Docker alternative: `docker-compose up` (Jekyll 4.2.2, ports 4000 + 35729).

There are no tests in this project.

## Architecture

Static portfolio site built with **Jekyll 4.3** (Ruby), deployed to GitHub Pages via GitHub Actions.

**Key files:**
- `index.html` — Single-page homepage with all sections (Hero, About, Experience, Education, Projects, Writing)
- `_config.yml` — Site metadata, social links, collections config, plugin config
- `assets/css/main.css` — All styling (~761 lines); dark theme with CSS custom properties, no CSS framework
- `_data/medium_posts.yml` — Auto-generated by CI daily; never edit manually

**Content structure:**
- `_projects/*.md` — One file per project; frontmatter drives cards and detail pages (title, slug, order, stack, stack_logos, repo_url, demo_url, image, card_bg)
- `_layouts/default.html` — Base layout (header/footer)
- `_layouts/project.html` — Project detail pages with tech badge rendering
- `_includes/` — `header.html` (nav + mobile toggle), `footer.html` (social links)

**Medium sync:** `scripts/fetch_medium.py` fetches the RSS feed, strips HTML, and writes `_data/medium_posts.yml`. GitHub Actions (`medium-sync.yml`) runs this daily at 06:00 UTC and auto-commits if the file changes.

**CI/CD:** Two workflows in `.github/workflows/`:
- `jekyll.yml` — Build + deploy to GitHub Pages on push to `main`
- `medium-sync.yml` — Daily Medium article sync
